# FatCatOS

Минимальная 32-битная операционная система с командной строкой и файловой системой в оперативной памяти.  
Загружается через GRUB, работает в текстовом режиме VGA и управляется с клавиатуры.

## Возможности

- Полноценная оболочка с поддержкой клавиатуры (скан-коды PS/2)
- Иерархическая файловая система в ОЗУ (каталоги и файлы)
- Команды: `dir`, `cd`, `mkdir`, `rmdir`, `echo`, `type`, `del`, `ren`, `copy`, `pwd`, `cls`, `help`, `parrot`
- Перенаправление вывода `echo` в файл (`>` и `>>`)
- Относительные и абсолютные пути, специальные каталоги `.` и `..`
- Простой скроллинг экрана и обработка backspace

## Структура проекта

```text
my_os/
├── src/
│   ├── boot.asm        # Мультизагрузочный заголовок и точка входа
│   ├── kernel.c        # Ядро: инициализация терминала, ввод/вывод
│   ├── shell.c         # Командная оболочка + реализация ФС
│   └── linker.ld       # Скрипт компоновщика
├── Makefile            # Сборка ядра и ISO-образа
├── grub.cfg            # Конфигурация загрузчика GRUB
└── README.md
```

## Требования для сборки

- ОС Linux (также работает в WSL)
- Установленные пакеты:
  - `gcc-multilib` (или просто `gcc`)
  - `nasm`
  - `ld` (GNU binutils)
  - `grub-pc-bin`, `xorriso`, `mtools`
- Для эмуляции: `qemu-system-i386`

Установка на Debian/Kali:
```bash
sudo apt update
sudo apt install nasm gcc make xorriso grub-pc-bin mtools qemu-system-x86
```

## Сборка и запуск

1. Клонируйте или создайте структуру папок, как показано выше, и поместите исходники в `src/`.
2. Выполните в корне проекта:
   ```bash
   make clean
   make all
   ```
   В папке `build/` появится `mini_os.iso`.
3. Запустите в эмуляторе:
   ```bash
   qemu-system-i386 -cdrom build/mini_os.iso
   ```
   Для запуска на реальном компьютере запишите ISO на флешку (например, через `dd` или Rufus), но **осторожно** – это перезапишет загрузочный сектор.

## Использование

После загрузки вы увидите приглашение командной строки, например:
```
/>
```
Файловая система уже содержит несколько тестовых папок и файлов:
- `/README.TXT`
- `/home/`
- `/home/user/HELLO.C`

### Список команд

| Команда | Описание |
|---------|----------|
| `help` | Показать справку по всем командам |
| `dir` | Содержимое текущего каталога |
| `cd <путь>` | Сменить каталог |
| `mkdir <имя>` | Создать новый каталог |
| `rmdir <имя>` | Удалить пустой каталог |
| `echo <текст>` | Вывести текст на экран |
| `echo <текст> > <файл>` | Записать текст в файл (перезапись) |
| `echo <текст> >> <файл>` | Добавить текст в конец файла |
| `type <файл>` | Показать содержимое файла |
| `del <файл>` | Удалить файл |
| `ren <старое> <новое>` | Переименовать файл или каталог |
| `copy <исх> <приём>` | Скопировать файл |
| `pwd` | Показать текущий путь |
| `cls` | Очистить экран |
| `parrot <текст>` | Попугай повторяет текст (для веселья) |
| `exit` | Выключить систему (остановка процессора) |

## Ограничения и TODO

- **Нет сохранения на диск.** Файловая система существует только в ОЗУ, все изменения теряются после выключения. Реализация драйверов ATA/USB и поддержки FAT32 – отдельный большой проект.
- **Максимальный размер файла:** 512 байт (ограничение буфера).
- **Общий размер кучи:** ~4 КБ, после чего `malloc` вернёт NULL.
- **Нет многозадачности, сети, графики.** Система однозадачная, работает в режиме ядра.
- **Клавиатура:** поддерживается только PS/2; если эмулятор настроен на USB-клавиатуру, может не работать. В QEMU по умолчанию эмулируется PS/2.
- **Нет поддержки строчных/прописных различий:** все имена чувствительны к регистру.
- **Нет проверок на корректность ввода** (например, нельзя удалить непустой каталог, но можно попытаться – получите сообщение об ошибке).

Проект создан в образовательных целях для понимания основ разработки ОС.
